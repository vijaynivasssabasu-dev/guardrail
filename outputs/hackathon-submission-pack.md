## Submission Description

Guardrail is a local-first trust layer for AI-generated code. Developers paste a unified diff or upload a patch, state the intended behavior, then Guardrail turns it into a review workspace that prioritizes risky changes in authentication, validation, data access, configuration, dependencies, payments, and tests. Its Intent Check exposes possible scope drift between the requested behavior and the actual code footprint. Each flagged hunk and scope concern must be acknowledged before approval, keeping the human reviewer responsible for the final decision.

Guardrail uses the OpenAI Responses API for concise review findings, personalized localized greetings, and developer questions about individual hunks when a credited API key is configured. It also remains fully testable without paid API access through clearly labelled deterministic fallbacks. History is stored in the browser with IndexedDB, allowing a visitor to re-audit prior diffs without server-side files, shared history, or Vercel write failures. Guardrail focuses on one practical job: making AI-assisted code changes easier to inspect before they ship.
